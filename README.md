项目概要介绍
一、项目名称
多模态AI互动式教学智能体
（Multi‑modal AI Interactive Teaching Agent）

二、参赛信息
赛题编号：A04

命题企业：锐捷网络

参赛团队：503小队

三、项目背景
当前，教育信息化已进入深度融合阶段，但教师在备课过程中仍面临显著痛点：

工具碎片化：课件制作、教案撰写、资源搜寻需在多平台间切换，流程割裂。

意图理解浅层化：现有AI工具只能执行简单指令，无法通过多轮对话深度把握教师的教学思路、重难点和个性化风格。

多模态资源利用低：教师已有的PDF、视频、图片等参考资料难以被自动解析并融入课件生成过程。

锐捷网络提出的“多模态AI互动式教学智能体”赛题，正是希望构建一个以教师教学思路为核心、具备深度对话与多模态生成能力的智能辅助系统，将教师从重复劳动中解放出来，回归“教学设计师”角色。

四、项目目标
开发一个智能、高效、个性化的课件共创系统，实现以下核心目标：

减负增效：将课件制作时间从数小时缩短至分钟级。

思路聚焦：让教师专注于教学设计与内容质量，而非格式排版。

深度个性化：通过多轮对话理解每位教师的独特教学风格和具体要求。

多模态融合：自动解析并利用PDF、Word、PPT、图片、视频等参考资料。

迭代优化：支持预览、修改、再生成的闭环流程。

五、核心功能
功能模块	具体能力
多模态交互输入	支持语音（Whisper）+文字两种方式；教师可上传PDF、Word、PPT、图片、视频等参考资料，并标注参考对应关系。
教学意图理解	通过多轮对话主动澄清教学目标、知识点、重难点、时长、风格等；利用大模型（阿里Qwen）结构化提取教学要素。
本地知识库RAG	构建教育专业知识库（教案、课程标准等），通过向量检索增强生成内容的专业性和准确性。
多模态内容解析	文本提取（PDF/Word/PPT）、OCR（PaddleOCR）、图像描述（BLIP）、视频抽帧分析（OpenCV+CLIP）。
课件生成引擎	自动生成PPT（python-pptx）、配套Word教案（python-docx），并可生成动画创意/互动游戏（HTML5/JS）。
迭代优化与导出	提供课件预览界面，支持自然语言修改意见（如“简化第三页”“增加案例”），增量修改后导出标准.pptx/.docx文件。

六、技术架构
前端（Web） ↔ FastAPI 后端 ↔ 核心服务层
                ├── 对话管理服务
                ├── 意图理解（Qwen）
                ├── RAG检索（Chroma + sentence-transformers）
                ├── 多模态解析（PaddleOCR, BLIP, OpenCV）
                └── 课件生成（python-pptx, python-docx, LLM生成HTML5）
关键技术选型：

大语言模型：阿里Qwen（通过DashScope API或本地部署）

语音识别：Whisper

向量数据库：Chroma

文本嵌入：BAAI/bge-large-zh

多模态理解：CLIP（图文匹配）、BLIP（图像描述）、PaddleOCR

视频处理：OpenCV（抽帧）

后端框架：FastAPI（Python 3.10）

课件生成：python-pptx、python-docx

七、创新亮点
深度教学意图理解：不同于单轮指令执行，通过多轮主动追问构建完整的教学需求画像。

多模态资料无缝融合：支持教师上传任意格式的参考资料，自动提取关键信息并融入生成过程。

RAG增强的专业知识：内置学科知识库，确保生成内容符合教学规律和课程标准。

创意内容自动生成：不仅生成PPT/教案，还能根据知识点自动设计互动小游戏（HTML5）或动画创意。

迭代式人机协作：教师预览后可随时提出修改，智能体精准理解并增量调整，形成闭环优化。

八、应用场景与价值
主要场景：中小学及高校教师日常备课、公开课准备、在线课程设计。

扩展场景：教师培训、数字化教学资源建设、自适应学习内容生成。

价值体现：

为教师：节省80%以上课件制作时间，提升教学创意质量。

为学生：获得更生动、个性化的学习材料。

为学校：推动教育数字化转型，沉淀优质教学资源。
